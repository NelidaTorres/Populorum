Push Notifications

One obvious thing after visiting events with multiple talks is that not every single one can be interesting to you.

You might spend much time chatting/networking with people outside the main room. And that's where Push Notifications can be helpful:

  • Attending an Event - A push notification will be sent to you a day before and 30 minutes before the Event's start.
  • Attending a Talk - A push notification will be sent 10 minutes before the talk starts. This lets you mark the talks you are interested in and get notified when they happen!

We would use Firebase and NativePHP Push Notifications.

Connecting with People

We have built a Profile page for our users, but it has no great use cases currently. We want to change that!

You might have met a new person at the Event, but exchanging details can be a hassle. Finding them on Social networks or LinkedIn takes your mind off things.

So instead, we want to build a feature where:

  • Each Event attendee would have their profile saved in a QR code
  • Add a built-in QR code scanner, which would add people to a specific Event
  • See a list of people you have met at each event!

This is also a fun game: you talk to as many attendees as possible and collect connections!

For this, we would use:

  • NativePHP Camera module - to trigger mobile phone camera
  • Remote File Upload - Sending the image to our server for parsing
