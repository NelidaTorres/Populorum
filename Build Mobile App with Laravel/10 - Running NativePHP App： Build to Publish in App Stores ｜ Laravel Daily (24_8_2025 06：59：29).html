Running NativePHP App: Build to Publish in App Stores

Lesson 10/11 Â· 2 min read

From a code perspective, we have a fully functional application.

But now we have a question: How do we run it in a "Test" environment and launch it in an app store?

Running Local Variants

Running the application locally is as simple as running:

Note: If your API is running locally, remember to change the .env URL to Ngrok or a public one.

php artisan native:run

If you have it connected, you can play with your application in an emulator or on a real device.

Here are a few guides on how to do that:

  • Android
  • iOS

For iOS, there's another option:

TestFlight - this allows you to simulate an App Store installation

Note: If you know any other ways to run apps, let us know!

Creating Publishable Builds

To create publishable builds of your application, you need to run two commands:

php artisan native:release patch # For application patches `0.0.X`
php artisan native:release minor # For minor releasese `0.X.X`
php artisan native:release major # For major releasese `X.0.0`

And as a first thing, we do have to tell that it's a major release:

php artisan native:release major

Then we can run a build:

php artisan native:run --build=release

Which will ask you for a platform. Once you have done this, it should start a build:

From there, you should be able to test the release version before going into publishing steps!

Note: You should also test this version on real devices before publishing

Publishing to App Stores (Google Play and App Store)

The final step in Mobile app Development is publishing to stores. For this, we and the NativePHP team will link you to official guides:

  • Google Play Store Guide
  • App Store Guide

There's no better/more up-to-date guide than the official ones!

Once you submit an application, it must pass the "Quality Assurance Proccess" before it will be verified. And in most cases, you will get detailed reports on bugs/issues. Once nothing is found, your app will be approved!
